This page gives a quick view of where TerraScale stands today and where the platform is headed next. For the full status view, see the [Public Roadmap](/roadmap/).

## Public Alpha

TerraScale is currently in Public Alpha. Teams can already build with the native API, the DynamoDB-compatible API, and the SQL interface. The platform also includes the C# SDK, multi-region replication, support across 19 global regions, and up to 100 GB of storage per database.

If you want the current stage in more detail, see [Public Alpha](/roadmap/public-alpha/).

## Public Alpha Roadmap

The next phase of Public Alpha is focused on reducing manual tuning and expanding capacity. Planned improvements include automatic indexing for frequently queried fields, automatic caching with invalidation, and an increase from 100 GB to 500 GB of storage per database.

Read the full plan on the [Public Alpha Roadmap](/roadmap/public-alpha-roadmap/).

## Stable Release

Stable release is about trust, not just feature count. The focus shifts to operational consistency, stronger reliability guarantees, and a smoother day-to-day developer experience.

See [Release Expectations](/roadmap/release/) for the criteria TerraScale needs to meet before it should be considered stable.

## How to Stay Updated

The roadmap evolves with real usage and feedback. Join the [Discord community](https://discord.gg/8Zr2Nw9g) to share requests and follow product direction, then check the [blog](/blog/) for deeper updates, technical writeups, and release context.

If you're new to the platform, start with the [Getting Started guide](/guides/getting-started/).
